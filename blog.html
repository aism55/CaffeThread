iOS Tutorials: Learn By Doing

March 30, 2014

As a newbie in iPhone development, one of my favourite sites to go to for tutorials is Ray Wenderlich.

It has a huge collection of written tutorials that shows step-by-step instructions for popular iOS topics. They provide clear screen shots as well as code snippets. After each code snippet, they explain what each line does and some brief theory on the iOS API. Another things that is cool about these tutorials is that users can download a starter project (rather than starting from scratch and creating the views and object c classes). Users can jump right in and follow the steps to complete the project. If you ever get stuck, they include a download link for the full completed project.

I am not a theory kind of person; I learn best from doing. It is not very motivating to have to read +600 pages book on iOS development. In the end, I end up giving up after the first two chapters! From Ray Wenderlichâ€™s site, I learn while doing. It is a lot more motivating to be able to create a working app (even if it is just copying and pasting code), than to wait weeks before diving to do actual coding. The site also sells ebooks on iOS development and keeps up to date to the latest versions.

Review: Pocket

February 25, 2014

I started using the iOS and android app call Pocket. Basically, Pocket is a service that allows users to save webpages/articles from the browser or news apps to be read later (even offline). It is synced across devices (iOS and android) and even has plugins for common web browsers such as chrome, safari and Firefox.

I use safari on my MacBook and was able install their bookmarklet (adds a small icon above the favorites bar) for quick saving of webpages. So far, I am pretty satisfied with the app and ease of use. Idea is simple, UI is clean and does its job. I can see myself potentially using Pocket to save articles to read for when I am in the metro. As a plus, Flipboard and many other apps supports Pocket, meaning I can save articles from Flipboard in my Pocket list.

Definitely worth it to checkout Pocket.

Managing Multiple Github Accounts

February 23, 2014

I've recently created a private github account but I didn't want to delete my current public github account. Hence I had two github accounts with two separate ssh keys. I couldn't push/pull from the private account since it didn't know which ssh key to use.

Finally found the solution from these two sites:

  • Code.Tutsplus- How to Work with Github and Multiple Accounts

  • Github/jexchan- Multiple SSH Keys Settings for Different Github Account


I will give a tutorial here since it was a combination of the two sites that gave me the correct settings. (Works for public or private accounts)

Create the ssh keys

Assuming we have two github accounts with their ssh keys under ~/.ssh

~/.ssh/id_rsa_personal
~/.ssh/id_rsa_work

We have a personal github account and a work account.

Create the config file

Under ~/.ssh , we need to create a file call config.

cd ~/.ssh/
vim config

Edit the config file as follow:

#Personal GitHub account
Host github.com-personal
HostName github.com
IdentityFile ~/.ssh/id_rsa_personal
IdentitiesOnly yes

# Personal GitHub account
Host github.com-personal
HostName github.com
IdentityFile ~/.ssh/id_rsa_personal
IdentitiesOnly yes

The Host can be named anything you want.

Cloning the git repository

Given that the ssh clone URL of your personal git repository is:

git@github.com:username/RepoName.git

Then in order to clone it, we need to replace the github.com with github.com-personal.

git clone git@github.com-personal:username/RepoName.git

Similarly, if the repository was from the work github, then it'll be:

git clone git@github.com-work:username/RepoName.git

Initializing an empty git repo

Given that we want to add existing local code to a new repository on github.

  1. Create an empty repo on github.
  2. On the computer under the project, we need to initialize the git. 
    git init
  3. Then do the normal git add and git commit commands: 
    git add .
    git commit -m "initial commit"
  4. Next, we need to set the remote repository to the one we just created. 
    git remote add origin git@github.com-personal:username/RepoName.git
  5. Lastly, push the first commit: 
    git push -u origin master
  6. From here onwards, you can do the normal commands 
    git push
